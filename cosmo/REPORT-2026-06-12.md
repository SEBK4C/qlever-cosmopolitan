# Bring-up report — 2026-06-12: first successful link

**TL;DR: the port works.** Starting from a completely bare Debian 12 server
(no compiler, no make, no cmake, no unzip, no sudo), the entire pipeline —
toolchain → deps sysroot → QLever build → link → smoke test → ICU
packaging — was rebuilt from scratch in one session. `qlever-index`,
`qlever-server` and `PrintIndexVersionMain` now exist as fat
x86_64+aarch64 Actually Portable Executables, with ICU data embedded.
This was the first time the final link was ever observed to succeed
(the previous server died after the abseil fix was written but before
its build finished).

## Where this picks up

Per the plan in `README.md`, work stopped mid-Phase-3 on the old machine
with one open question: does the abseil `ABSL_HAVE_MMAP` patch
(`patch_absl_cosmo.sh`) clear the last undefined-symbol wall
(`absl::synchronization_internal::CreateThreadIdentity` /
`LowLevelAlloc`)?

**Answer: yes.** No further undefined-symbol rounds appeared. The
README's prediction — that any new symbols would be the same
"dep self-disables under `__COSMOPOLITAN__`" pattern — was not needed.

## What was done, in order

1. **Toolchain** (`~/cosmocc`): downloaded current cosmocc (GCC 14.1.0).
   The host has no `unzip`, so extraction used python `zipfile` — which
   silently destroyed all 38 symlinks and produced a spectacularly
   misleading `cosmoc++` failure. Root-caused and fixed (AUDIT quirk #1).
2. **Host bootstrap, no sudo**: `make` = cosmocc's GNU Make 4.4.1,
   `assimilate -x`'d to native ELF (CMake can't spawn APE binaries);
   `cmake` = Kitware 3.31.7 binary tarball (apt's 3.25 < required 3.27)
   (AUDIT quirk #4).
3. **Sysroot** (`build-deps.sh`): zlib, zstd, OpenSSL built clean. ICU
   failed twice:
   - first on the broken-symlink `cosmoc++` (quirk #1),
   - then on a genuine latent bug: ICU's `common/Makefile` regenerates
     *itself* mid-build (via its `svchook.mk` self-prerequisite),
     resurrecting the quoted `-D` defines that `build-deps.sh` had
     sed-stripped — the quoted-args failure the sed exists to prevent.
     It had only ever worked on the old server because reruns inherited
     a leftover `svchook.mk`. **Fixed durably in `build-deps.sh`**
     (quirk #2).
   Boost then failed at `bootstrap.sh` — b2 needs a host C++ compiler,
   which doesn't exist here. **Fixed durably in `build-deps.sh`**: b2 is
   built with `cosmoc++` and assimilated when no host compiler is found
   (quirk #3). After that, all of Boost built; all 13 archives have their
   `lib/.aarch64/` twins; no warnings.
4. **QLever build** (`build.sh`): configure + FetchContent (abseil with
   the PATCH_COMMAND, ANTLR4, re2, s2, spatialjoin, fsst, nlohmann-json)
   all clean. Compile reached 84% at `JOBS=5`, then the OOM killer took
   out cc1plus on the `sparqlExpressions` TUs (this host has ~10 GB free —
   a KVM VM owns the rest). Finished at `JOBS=2`, then `JOBS=1` for the
   heaviest TUs (quirk #5).
5. **Link**: `qlever-index` (113 MB), `qlever-server` (117 MB),
   `PrintIndexVersionMain` (2.9 MB) — all linked on the first attempt
   with the abseil fix in place.
6. **Smoke tests**: `PrintIndexVersionMain` prints its version JSON;
   `qlever-server --help` prints full options (program_options, ICU init
   path exercised).
7. **Packaging** (`package.sh`): `icudt76l.dat` embedded into the zip
   section of both binaries via the cosmos Info-ZIP.
